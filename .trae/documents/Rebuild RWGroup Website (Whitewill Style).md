# Page Design Spec (desktop-first) — RWgroup

## Global Styles

* Layout: CSS Grid для сеток (карточки/галереи), Flexbox для горизонтальных групп (хедер/фильтры/CTA).

* Breakpoints: Desktop ≥1200, Tablet 768–1199, Mobile <768 (на мобилке фильтры в drawer, меню якорей в burger).

* Tokens: фон #FFFFFF, текст #0F172A, вторичный текст #475569, акцент #0EA5E9, границы #E2E8F0.

* Typography: H1 48/56, H2 32/40, H3 24/32, Body 16/24, Small 14/20.

* Buttons: Primary (акцент, белый текст), Secondary (обводка), Hover: +5% темнее, Focus ring.

* Links: underline on hover.

## Meta (общие правила)

* Title: «RWgroup — агентство недвижимости» + суффикс страницы.

* Description: 140–160 символов, с ключами «недвижимость Москва», «новостройки/вторичка/аренда».

* OpenGraph: og:title/description, og:image (hero или cover), og:type=website.

***

## 1) Главная (/)

**Page structure:** лендинг из 10 экранов, вертикальный стек, якорная навигация.

* Header (sticky): логотип слева; контакты (телефон кликабелен); якоря; CTA «Получить консультацию».

* Экран 1 Hero: слева текст/CTA (2 кнопки), справа/фон — панорама; блок доверия как badge.

* Экран 2 Каталоги: Tabs (3); сверху фильтры (desktop в одну строку, wrap), снизу grid карточек (3 колонки).

* Экран 3 Лучшие предложения: 2–3 секции-карусели (ЖК/Лоты/Подборки) — контент из админки.

* Экраны 4–10: карточки преимуществ (grid), стоимость услуг (2–3 блока + CTA), этапы (stepper), миссия (текст), команда (2 колонки: фото/текст), отзывы (cards), партнёрство (текст+форма).

* Footer: контакты, соцсети, блог-ссылка, политика.

**Forms placement:**

* CTA «Консультация» (форма #1) в Hero/хедере.

* CTA «Купить/Продать/Сдать» в блоке каталогов (форма #2 с вкладками).

* «Стать партнёром» (форма #4) в экране 10.

***

## 2) Каталог недвижимости (/catalog)

**Layout:** 2 колонки на desktop (слева фильтры, справа листинг), на mobile фильтры в drawer.

* Tabs: Новостройки/Вторичка/Аренда.

* Filters: спальни (select), цена (min/max), район, метро, поиск по названию ЖК (только для новостроек).

* Listing: карточки (image, title, район/метро, спальни, площадь, цена, CTA «Узнать детали»).

* States: empty (нет результатов), loading (skeleton), error.

***

## 3) Карточка лота (/property/\[slug])

**Layout:** 2 колонки на desktop.

* Gallery: большая картинка + превью (grid), fallback при отсутствии.

* Summary: title, deal\_type, цена (+ период для аренды), спальни, площадь, район/метро, статус.

* CTA: кнопка «Записаться на просмотр / Узнать детали» (форма #3 в модалке).

***

## 4) Карточка ЖК (/complex/\[slug])

**Layout:** header-блок ЖК + табличный/карточный список лотов ниже.

* Top: title, район/метро, price\_from/area\_from, статус, медиа; (опционально) мини-карта по geo\_lat/lon.

* Lots list: фильтруемый список лотов ЖК (минимум: спальни/цена).

* CTA: форма #3.

***

## 5) Подборка (/collection/\[slug])

**Layout:** обложка + описание + список items.

* Cover: image + title overlay.

* Items: grid карточек (с бейджем типа: «Лот»/«ЖК»), переход в соответствующую карточку.

***

## 6) Политика (/privacy)

**Layout:** один контейнер 720–840px, типографика для длинного текста.

* Вставить блок согласия на ПДн (текстовая ссылка используется во всех формах).

***

## 7) Админ-панель (/admin)

**Layout:** dashboard (sidebar слева, контент справа).

* Разделы: «Витрина (экран 3)», «Подборки», «Фиды», «Импорт (маппинг/предпросмотр)», «Логи импорта», «Лиды».

* Импорт: шаги (выбор источника → маппинг → предпросмотр → запуск), таблица ошибок.
